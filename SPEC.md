## 1. 問題陳述

### 1.1 目標使用者

| 族群 | 規模 | 痛點 |
|---|---|---|
| 多平台電商賣家（蝦皮 + Yahoo + Momo + PChome） | ~5 萬 | 每個平台都要手動上架、重複填寫同樣商品資料 |
| 自有官網 + 多平台賣家 | ~2 萬 | 商品主檔分散在不同平台後台，更新價格/庫存要 4 處同步 |
| 團購主/小型品牌 | ~3 萬 | 想跨平台擴展但沒工程師、不會寫 API 串接 |

### 1.2 為什麼不做替代方案

- **各平台後台逐一上架**：耗時 15-30 分鐘/商品 × 4 平台 = 1-2 小時
- **多平台 ERP 系統（Shopline/Bnext）**：月費 1,500-5,000 NT$，對微型賣家太貴
- **自己寫 API 串接**：需工程師、每平台 API 規格不同、維護成本高
- **我們的解法**：商品主檔一次填，CSV 批次產出 5 平台格式，零月費

---

## 2. 解決方案

### 2.1 核心價值主張

> 「商品資料填一次，5 個平台 CSV 一次匯出。」

### 2.2 使用者流程

1. 建立商品主檔（標題/價格/庫存/圖片/描述/規格）
2. 套用平台標題公式（例：「蝦皮版：商品名 + 促銷詞」「Momo 版：品牌 + 型號」）
3. 圖片自動 resize 到各平台規格（蝦皮正方形 800x800、PChome 800x800 等）
4. 描述 Markdown → 各平台格式轉換（HTML/純文字/Markdown）
5. 一鍵下載 ZIP（內含 5 平台 CSV + 處理好的圖片）

## 3. 功能清單

### 3.1 MVP（必做）

- [ ] 商品主檔 CRUD（標題/價格/庫存/SKU/分類/圖片）
- [ ] 5 平台 CSV 適配器（蝦皮/露天/PChome/Yahoo/自家站）
- [ ] 圖片自動 resize（依平台規格）
- [ ] 標題公式引擎（變數插值 + 平台前綴後綴）
- [ ] 描述格式轉換（Markdown ↔ HTML ↔ Plain Text）
- [ ] 庫存預警（低於設定閾值提示）

### 3.2 v2（加值）

- [ ] 各平台 API 自動上架（需平台授權）
- [ ] 訂單統一管理（從 5 平台抓訂單整合）
- [ ] 促銷檔期排程
- [ ] 跨平台價格同步

### 3.3 明確不做

- 金流收款（交給各平台處理）
- 物流配送整合（先做檔案產生、不做 API 對接）
- 多語系商品（先做繁中）
- 跨境電商平台（Amazon/eBay 等）

## 4. 技術棧

| 層 | 選擇 | 理由 |
|---|---|---|
| 前端 | React 18 + Vite + TypeScript | 與既有架構一致 |
| CSV 產生 | Papa Parse (unparse) | 業界標準 |
| 圖片處理 | browser-image-compression + Canvas | 純前端、無需後端 |
| ZIP 打包 | JSZip | 純前端 ZIP 產生 |
| 資料持久化 | localStorage |
| 部署 | Vercel |

## 6. 風險與決策

| 風險 | 等級 | 緩解 |
|---|---|---|
| 平台 CSV 格式變更導致無法上架 | 🔴 高 | 適配器模組化、定期追蹤平台公告、提供檢查工具 |
| 圖片版權爭議（上傳他人圖片） | 🟠 中 | 明確聲明使用者需自負；提供浮水印工具 |
| 商品資料外洩（裝置被駭） | 🟡 低 | localStorage 加密（v2） |
| 庫存同步延遲（多平台同時上架同一商品超賣） | 🟠 中 | 提供「平台庫存互斥」設定、警示超賣風險 |
